# 技術選定

本プロジェクトで採用する技術スタックの **採用理由（Why）** を保管する。具体的なバージョン・依存ライブラリ一覧は `package.json` が情報源とし、本ドキュメントには書かない。

レイヤ構成・エラー伝搬・モック方式は `./overview.md` を参照。

---

## 1. 言語・フレームワーク

### TypeScript

型安全性と AI によるコード理解の精度向上のために必須。`unknown` 由来の動的データは valibot で必ずパースする。

### React Router v7（Framework mode）

- **採用理由**: Remix のフルスタック機能を受け継ぎつつ、SPA と SSR を柔軟に切り替え可能。Cloudflare Pages へのネイティブ対応が強力。
- **役割**: ルーティング、BFF 層（loader / action）、UI コンポーネントの管理。

---

## 2. 実行環境

### Cloudflare Pages / Workers

- **採用理由**: グローバルなエッジランタイムにより高い応答速度を実現。無料枠内での運用が可能。
- **制約**: `workerd` ランタイム上で動くため、Node.js 固有 API（`crypto`, `fs` など）への依存を避け、常に Web 標準 API（`fetch`, `crypto.subtle`）を利用する。

---

## 3. 静的解析・フォーマッター

### Biome

- **採用理由**: ESLint と Prettier を統合した高速ツール。設定が簡潔であり、AI エージェントがプロジェクトのコーディング規約を把握する際のノイズを最小限に抑える。

---

## 4. バリデーション・型定義

### Valibot

- **採用理由**: Zod と比較してランタイム時のバンドルサイズを大幅に削減できる（Tree-shaking 対応）。エッジランタイムの制限に抵触しにくく、型推論能力も同等。
- **役割**: スプレッドシートからの取得データ、LINE Webhook のペイロード、LIFF フォーム入力のバリデーション。型は `InferOutput` で導出する。

---

## 5. セキュリティ・認証

### jose

- **採用理由**: Node.js の `crypto` モジュールに依存せず、Web 標準の Web Crypto API で動作する軽量 JWT ライブラリ。
- **役割**: Google Service Account の JWT 生成・署名、および LINE ID トークンの検証。

---

## 6. テスト・品質保証

### Vitest

- **採用理由**: Vite ベースの React Router v7 と相性が良く、非常に高速。
- **役割**: 意図抽出パーサーや計算ロジックなどのドメインロジックの単体テスト。

### Playwright

- **採用理由**: 実際のブラウザ環境での挙動を検証。
- **役割**: LIFF の表示確認、モック環境下でのエンドツーエンドテスト。

テスト戦略の詳細は `.claude/skills/testing/` を参照。

---

## 7. UI・デザインシステム

### Tailwind CSS

- **採用理由**: AI コーディングエージェントが最も得意とする CSS フレームワークであり、スタイル生成の精度が高い。

### shadcn/ui

- **採用理由**: ソースコードがプロジェクト内に直接配置されるため、AI がコンポーネントの実装詳細を理解し、カスタマイズしやすい。

UI 言語と運用規約は `.claude/skills/design-system/` を参照。

---

## 8. 外部 API 連携（SDK 非依存）

### Fetch API

- **採用理由**: Google 公式 SDK の依存関係（Node.js 固有 API）を排除し、エッジ環境で確実に動作させるため。
- **方針**: Google Sheets API および Gemini API との通信はすべてピュアな REST リクエストで行う。

---

## 9. Webhook / LIFF の検証手段

### モックスクリプトによる擬似 Webhook

- **手法**: 用意したモック JSON（レシート送信イベント等）をローカルのエンドポイントへ直接 POST するスクリプトを実行（`scripts/mock-webhook.sh`）。
- **理由**: 開発用スプレッドシートへの書き込みやバリデーションロジックの挙動を、外部環境に依存せず高速に確認するため。

### LIFF モック機能

- **手法**: LIFF モック機能を有効化し、PC のブラウザ上でダッシュボードの表示を確認。
- **理由**: 実機での検証回数を減らし、フロントエンドの表示崩れやステート管理のデバッグを効率化するため。

具体的な起動手順は `README.md` を参照。
