# kakeibot — エージェント向け指示

kakeibot は LINE 上で動く家族共有の家計簿ボット。Cloudflare Workers + React Router v7 + Google Sheets。

## 言語

応答・ドキュメント・コード内コメント・エラーメッセージ・プレースホルダーはすべて日本語で記述する。

## 第一原則: まず Skill を見る

実装・設計・レビューに着手する前に、必ず `.claude/skills/` 配下の Skill を探索する。Skill はこのプロジェクトの規約・ドメイン語彙・暗黙ルール・参照先マップを束ねた索引であり、コードに表せない決定の入口。タスクに関係しそうな Skill が 1 つでもあれば、コードに触れる前に読む。

## ディレクトリの役割

- `app/` — 実装の本体。コードベースを現在の正確な仕様（単一情報源）として信頼する。
- `docs/` — 過去の決定とコードに表せない状態（Why / 確定事実）の保管所。通常は Skill から誘導されて辿る。
- `README.md` — セットアップ・mock の 3 軸切り替え・dev 起動・テスト実行（人間とエージェントの共用）。開発環境を立ち上げる作業ではここを参照。
- `temp/` — 作業用バッファ。思考プロセスやドラフトに自由に使ってよい。

## 振る舞いの制約

- 破壊的な git 操作（`reset --hard` / `push --force` / ブランチ削除など）はユーザーの明示的許可がない限り行わない。
- pre-commit / pre-push hooks を `--no-verify` で迂回しない。失敗の根本原因を直す。
- ユーザーが明示的に依頼しない限り、新しい Markdown ドキュメントを増やさない。
